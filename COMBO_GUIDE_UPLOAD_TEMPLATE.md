# Шаблон для `/combo publish`

Этот файл - быстрый шаблон для подготовки двух `.txt` файлов:

- `combo_file`
- `techs_file`

Бот читает их не как свободный текст, а по довольно строгим правилам. Если отступить от шаблона, часть контента просто не распознается.

## Главное

- Имя персонажа должно совпадать в обоих файлах 1:1.
- Заголовок персонажа должен быть в формате `😀 **Name**`.
- Если нужен пинг роли, используй именно `- <@&ROLE_ID>` в исходнике.
- Не используй слишком длинный emoji в заголовке персонажа: парсер ожидает короткий emoji.
- Все, что стоит выше первого распознанного персонажа, бот считает черновым preamble и не публикует.
- В `techs.txt` сначала должен быть блок `🛠️ **Общие техи**`, а уже потом персонажные секции.
- Названия техов в `combo.txt` должны совпадать с названиями в `techs.txt`, иначе ссылки не подставятся.

## Точные Имена

В самом `combo.txt` бот не проверяет имя персонажа против отдельного справочника, но для нормальной интеграции во всю систему лучше использовать канонические имена из `bot.config.json`.

## `combo.txt`

Шаблон одного персонажа:

```txt
⚪ **Honored One**

**M1 + M1 + Uppercut**
> • Техи: Rapid Kick, RRTP
> **Урон:** 71.25

**Dash + M1 + Downslam**
> • Техи: Blue Downslam
> **Урон:** ~79.75
```

Правила для `combo.txt`:

- Персонаж начинается строкой вида `emoji **Имя**`.
- Дополнительно можно добавить роль: `emoji **Имя** - <@&1234567890>`.
- Комбо начинается с любой жирной строки, где есть ` + `.
- Блок комбо можно продолжать несколькими строками.
- Строка `> • Техи: ...` нужна, если хочешь, чтобы техи стали кликабельными ссылками после публикации.
- Техи в одной строке разделяй запятыми.
- Строка `> **Урон:** ...` подхватывается как damage.
- Если нужен именно текущий канонический персонаж, подставляй его `label` из таблицы выше без сокращений.

## `techs.txt`

Шаблон с общими и персонажными техами:

```txt
🛠️ **Общие техи**

**Новичок**
**Rapid Kick**
Короткое объяснение, когда и зачем нажимать.

**Средний уровень**
**RRTP**
Пошаговое описание и важные тайминги.

**Продвинутый уровень**
**Kick 'n Slam**
Развёрнутый разбор, если тех сложный.

⚪ **Honored One**

**Новичок**
**Blue Downslam**
Описание конкретного теха для Honored One.

**Средний уровень**
**Technique Extension**
Ещё один тех для этого персонажа.
```

Правила для `techs.txt`:

- Блок `🛠️ **Общие техи**` должен быть первым распознанным разделом.
- После него можно писать персонажные секции.
- Уровни допускаются только такие:
  - `**Новичок**`
  - `**Средний уровень**`
  - `**Продвинутый уровень**`
- Название теха должно быть отдельной жирной строкой: `**Название**`.
- Описание теха может быть многострочным.
- Не ставь важную информацию в preamble до первого распознанного раздела - бот это не публикует.
- Имена персонажей здесь должны совпадать с `label` из `bot.config.json`, если хочешь, чтобы техи и ссылки собрались без сюрпризов.

## Как заливать

### Полная публикация

```text
/combo publish
```

Нужны:

- `combo_file`
- `techs_file`
- `channel`

### Добавить одного персонажа

```text
/combo add
```

Нужны:

- `combo_file`
- `techs_file`

## Быстрый Путь

Если нужен новый персонаж для Crew Russia и хочется быстрее всего протащить его через всю систему, делай так:

1. Добавь персонажа в `bot.config.json` в `characters` с устойчивым `id` и каноническим `label`.
2. Добавь картинку в `assets/characters/<id>.png`, чтобы комбо-гайд мог показать изображение.
3. Если нужен Discord custom emoji, положи файл в `assets/character-emojis` и потом загрузи его через штатный flow.
4. Если старые/русские названия должны тоже матчиться в SoT и role sync, добавь алиасы в `src/sot/character-aliases.js`.
5. После этого публикуй комбо-гайд через `/combo publish` или точечно добавляй персонажа через `/combo add`.

Минимум для самого комбо-гайда:

- `combo.txt`
- `techs.txt`
- изображение персонажа в `assets/characters`

Минимум для всей системы персонажа:

- `bot.config.json`
- `assets/characters`
- `src/sot/character-aliases.js` при необходимости русских/старых имен
- Discord role / emoji, если они нужны в сообщениях и панелях

## Быстрый чек перед загрузкой

- В `combo.txt` каждый персонаж начинается новым заголовком `emoji **Name**`.
- В `techs.txt` есть `🛠️ **Общие техи**`.
- Названия персонажей совпадают между файлами.
- Названия техов в `> • Техи:` совпадают с `**Название теха**` в `techs.txt`.
- Нет важного текста выше первого распознанного блока.
- Файл не перегружен, чтобы комбо и техи не упирались в лимиты сообщений.

## Что бот делает сам

- Режет длинные комбо и техи на несколько сообщений.
- Подставляет ссылки на техи в блоках `> • Техи:`.
- Создаёт отдельную ветку для техов каждого персонажа.
- Делает общий тред для `Общие техи`, если они есть.

## Если что-то не так

- Если персонаж не распознался, проверь emoji в заголовке и формат `**Имя**`.
- Если техи не ссылаются друг на друга, проверь точное совпадение названий.
- Если персонажные техи не подхватились, проверь, что в `techs.txt` есть блок `🛠️ **Общие техи**` до персонажной секции.
- Если роль не подставляется, проверь, что в исходнике стоит именно `- <@&ROLE_ID>`.
- Если нужен новый персонаж в SoT / tierlist / role-sync, сначала проверь, что его `id` есть в `bot.config.json`, а потом уже заводи контент гайда.
