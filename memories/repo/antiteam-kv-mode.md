- KV ("КВ") is a 4th danger mode shown in the level dropdown (buildLevelSelect adds value "kv"); picking it flips the draft to kind="kv" and clears the author's Roblox (handleSelectMenuInteraction). It is its own ticket kind (ANTITEAM_TICKET_KINDS) and an anchor kind (ANTITEAM_ANCHOR_KINDS) alongside "clan".
- KV requires: anchorUserId + anchorNote (who/why important) + anchor Roblox. Anchor is collected via the at:kv_anchor:modal (mention/ID + reason); the modal auto-pulls the anchor's verified profile Roblox, else the author sets it with the twink button. getDraftPublishError gates publish on all three.
- KV publishes with status "pending_approval" ("ВОЗМОЖНО КВ"): finalizeTicketPublish pings ONLY the approval targets (config.kvApprovalRoleId/UserId, default KV_APPROVAL_PING_ROLE_ID=1519762809066361037 / KV_APPROVAL_PING_USER_ID=1011666449963688027), no battalion ping, no battalion-role grant, and the thread panel shows admin-only Approve/Reject.
- Admin (hasAdmin only) approve -> setKvApprovalDecision "approved" -> status "open", sendKvOpenPing does a REAL message ping of editPingRoleIds (edit-test) that actually notifies, thread opens for help. Reject -> "rejected" -> status "cancelled", finalizeClosedTicket locks/archives, no help, no points.
- pending_approval is in listOpenAntiteamTickets (so messages sync) but excluded from idle auto-close (findIdleAntiteamTickets skips kind kv and non-open status). isTicketContentEditable lets the author fix the direct link / anchor-Roblox while still pending.
